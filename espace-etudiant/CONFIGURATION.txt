ESPACE ETUDIANT — Configuration Supabase
========================================

⚠ NE PAS OUVRIR LE SITE EN file:/// (double-clic sur le dossier dans l’explorateur)
  Le navigateur affiche alors une « liste de fichiers » au lieu de la page, et
  la connexion Supabase + les redirections (/admin.html, etc.) ne fonctionnent pas.

  À la place :
  - Production : https://www.studyalready.com/espace-etudiant/
  - En local : lancer un petit serveur HTTP à la racine du projet StudyAlready, ex. :
      cd Desktop\Studyalready
      npx --yes serve -p 8000
    puis ouvrir http://localhost:8000/espace-etudiant/
    (ou ouvrir directement …/espace-etudiant/index.html dans la barre d’adresse
     après avoir démarré le serveur — jamais file:///C:/…)

✅ TERMINÉ : Supabase est déjà configuré dans assets/js/config.js
   (SUPABASE_URL + SUPABASE_ANON_KEY, projet "nevdhyekybmtvejhwhxz").

Si l'espace étudiant affiche un bandeau d'erreur de connexion :
  - Forcer le rechargement de la page : Ctrl + Shift + R
  - Tester en navigation privée
  - Vérifier que la console DevTools (F12) n'affiche pas d'erreur
    de blocage réseau (bloqueur de pub, firewall d'école…)

À vérifier une fois côté Supabase (Dashboard Supabase) :
  1) Authentication → Providers → Email
     - "Email" activé.
     - "Confirm email" : ACTIVÉ en production (sinon n'importe qui peut
       créer un compte sans vérification).
  2) Authentication → URL Configuration
     - Site URL : https://www.studyalready.com
     - Redirect URLs (ajoutez chacune) :
         https://www.studyalready.com/espace-etudiant/dashboard.html
         http://localhost:8000/espace-etudiant/dashboard.html  (tests locaux)

Tables utilisées par le site (déjà créées) :
  - public.profiles            (annuaire — créer-profil.html)
  - public.form_submissions    (tous les autres formulaires)
  - public.admins              (dashboard admin /admin.html)

Sécurité :
  - La clé "anon" est PUBLIQUE par design (côté navigateur).
  - Ne JAMAIS poser la clé "service_role" dans le site web.
  - Toutes les tables ont des Row Level Security (RLS) actives.
